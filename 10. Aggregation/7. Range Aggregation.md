# Range Aggregation

- This is for specifying a range so that the aggregation to bucket happens based on the range:

![RANGE][RANGE]

<br>
Query is written like this:
```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "my_agg": {
      "range": {
        "field": "total_amount",
        "ranges": [
          {
            "to": 50
          },
          {
            "from": 50,
            "to": 100
          },
          {
            "from": 100
          }
        ]
      }
    }
  }
}
```
which gives:
```
{
  "took" : 10,
  "timed_out" : false,
  "_shards" : {
    "total" : 1,
    "successful" : 1,
    "skipped" : 0,
    "failed" : 0
  },
  "hits" : {
    "total" : {
      "value" : 1000,
      "relation" : "eq"
    },
    "max_score" : null,
    "hits" : [ ]
  },
  "aggregations" : {
    "my_agg" : {
      "buckets" : [
        {
          "key" : "*-50.0",
          "to" : 50.0,
          "doc_count" : 164
        },
        {
          "key" : "50.0-100.0",
          "from" : 50.0,
          "to" : 100.0,
          "doc_count" : 347
        },
        {
          "key" : "100.0-*",
          "from" : 100.0,
          "doc_count" : 489
        }
      ]
    }
  }
}
```
We can see that the range is specified in each of the buckets in `buckets` array<br>
The same query can be written to have bucket names in the result using `keyed` and `key` parameters in the `range` and each entry in `ranges` array:
```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "my_agg": {
      "range": {
        "field": "total_amount",
        "keyed": true, 
        "ranges": [
          {
            "to": 50
            , "key": "one"
          },
          {
            "from": 50,
            "to": 100,
            "key": "two"
          },
          {
            "from": 100,
            "key": "three"
          }
        ]
      }
    }
  }
}
```
This gives us:
```
{
  "took" : 3,
  "timed_out" : false,
  "_shards" : {
    "total" : 1,
    "successful" : 1,
    "skipped" : 0,
    "failed" : 0
  },
  "hits" : {
    "total" : {
      "value" : 1000,
      "relation" : "eq"
    },
    "max_score" : null,
    "hits" : [ ]
  },
  "aggregations" : {
    "my_agg" : {
      "buckets" : {
        "one" : {
          "to" : 50.0,
          "doc_count" : 164
        },
        "two" : {
          "from" : 50.0,
          "to" : 100.0,
          "doc_count" : 347
        },
        "three" : {
          "from" : 100.0,
          "doc_count" : 489
        }
      }
    }
  }
}
```
We can see that the buckets have keys that we specified in the `ranges` array.

# Date Range Aggregation

- Conceptually the same thing. It works for dates.
```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "my_agg_x": {
      "date_range": {
        "field": "purchased_at",
        "ranges": [
          {
            "from": "2016-01-01",
            "to": "2016-01-01||+6M"
          },
          {
            "from": "2016-01-01||+6M",
            "to": "2016-01-01||+1y"
          }
        ]
      }
    }
  }
}
```

We can add `stats` aggregation to retrieve some more aggregation:
```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "my_agg_x": {
      "date_range": {
        "field": "purchased_at",
        "keyed": true, 
        "format": "yyyy-MM-dd", 
        "ranges": [
          {
            "from": "2016-01-01",
            "to": "2016-01-01||+6M",
            "key": "first half"
          },
          {
            "from": "2016-01-01||+6M",
            "to": "2016-01-01||+1y",
            "key": "second half"
          }
        ]
      },
      "aggs": {
        "my_agg_ou": {
          "stats": {
            "field": "total_amount"
          }
        }
      }
    }
  }
}
```
